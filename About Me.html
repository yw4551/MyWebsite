About Me

Hi, my name is Yitzi Weiner, and I’m a junior QA engineer. I am familiar with both types of testing: manual testing and automation testing. I can run tests on all types of platforms and services, like web and mobile testing. I can run tests on the client side and on the server side.
I have studied a high-level course of QA in the College of Automation by Gal Matalon (www.automation.co.il/). The course was split into two parts. The first part was the manual testing, and we learned how to prepare for the testing by writing the STP and STD documents, writing the test cases, and how to perform the tests and write the STR document.
The second part of the course was about automation testing, starting with learning the Java programming language, and then we learned how to write code in Selenium and use the most recent tools to run automation tests, like Maven to run the tests from the command line, Jenkins to run a test in advance, environment testing with Docker, and all of the new and most recent tools that you can use for automation testing.
My astrology When I come to check the product, the most important thing is to understand what this product is, so the first thing I will do is spend some time on the product and try to understand the application, and if there are any questions, I will send them to the customer until I understand the total application. Only then will I start writing the STP document and the STD document, and only like that can you be sure that you covered the whole application.
I believe that the tests should be executed with priority; the most important functions will be tested first, and then we will test more according to the time that is left. The priority that I will give will be based on whether it’s a show stopper, so it will get a high priority, and that will be the first thing we will be checking, and the next thing will be a function that doesn’t have a workaround that meets that this is the only way to use this feature, and then comes a feature that has a workaround, and after that, all the other features, and lastly, spelling mistakes and GUI bugs with the lowest priority.
All of the bugs must be tracked in a bug tracker program like Jira and TestRail, and I know how to use these tools. If a company uses another program, I’m happy to learn more about it and use it.
